What I Offer

Offered Services

Secure Code Review (SAST)

A comprehensive white-box audit of source code to identify architectural flaws and vulnerabilities before deployment. By analyzing the application from the inside, I uncover deep-seated issues that automated scanners miss.

Web Application Penetration Testing

Simulating real-world attacks to identify vulnerabilities such as SQL Injection, Cross-Site Scripting (XSS), and Broken Access Control. This ensures your web-facing assets are resilient against unauthorized access.

API Security Assessment

Specialized testing for RESTful and SOAP APIs to ensure secure data exchange. This includes testing for insecure endpoints, broken object-level authorization (BOLA), and improper authentication.

Infrastructure & Network Audit

Stress-testing network configurations, firewalls, and servers. I identify potential entry points and misconfigurations that could allow an attacker to pivot through your internal or external network.

Architectural Threat Modeling

Evaluating your system design from an engineering perspective. I identify structural security risks early in the design phase to help you build security into the foundation of your product.

Testing Standards & Frameworks

To ensure the highest level of rigor and compliance, all assessments are conducted in accordance with globally recognized cybersecurity standards. This ensures that the results are consistent, professional, and audit-ready.

OWASP Top 10 & ASVS

The primary framework used for Web and API assessments. I follow the OWASP Top 10 to identify the most critical risks, and utilize ASVS for deep-dive technical requirements and testing depth.

CWE/SANS Top 25

For Secure Code Reviews, I utilize the Common Weakness Enumeration (CWE) to identify the most prevalent and impactful software vulnerabilities at the development level.

NIST SP 800-115

My methodology for network and infrastructure audits aligns with the NIST Technical Guide to Information Security Testing and Assessment, providing a structured and repeatable approach.

CVSS v3.1

Every identified vulnerability is assigned a score based on severity, impact, and exploitability, giving your team a transparent and standardized remediation priority.
